# 평가 케이스: 감정 입력 → AI 피드백 응답

> **파일 위치**: `evals/emotion-input-eval.md`
> **목적**: 감정 입력 시 AI 피드백이 올바르게 생성되는지 검증

---

## 테스트 케이스 1: 기본 감정 입력

```
입력값:
  감정: "기쁨"
  강도: 8
  오늘의 다짐: "운동 가기"

기대출력:
  - AI 피드백에 공감 표현이 포함될 것
  - "운동"에 대한 격려 멘션이 포함될 것
  - 3-4문장 이내일 것
  - 한국어로 출력될 것
  - {"feedback": "...", "suggestedColor": "#FFD700"} 형식
```

## 테스트 케이스 2: 감정 전환 애니메이션

```
입력값:
  이전 감정: "분노" (#E74C3C)
  현재 감정: "평온" (#2ECC71)

기대출력:
  - 600ms 이내 전환 완료
  - 중간 색상 보간(interpolation)이 자연스러울 것
  - Orb 크기 변화가 부드러울 것 (급격한 점프 없음)
  - 햅틱 피드백이 한 번만 발생할 것
```

## 테스트 케이스 3: 오프라인 모드

```
조건:
  - 네트워크 연결 없음
  - 이전에 저장된 감정 데이터 있음

기대출력:
  - 앱이 정상 실행될 것
  - 로컬에 감정 데이터가 저장될 것
  - 네트워크 복구 시 자동 동기화될 것
  - 사용자에게 "오프라인 모드" 표시
```

## 테스트 케이스 4: 시간대별 화면 전환

```
입력값:
  현재 시간: "21:30"

기대출력:
  - 메인 화면이 나이트 모드로 자동 전환
  - "오늘 하루는 어땠나요?" → "오늘 하루를 마무리하며..."로 문구 변경
  - 배경색이 더 어두운 톤으로 변경
  - 강도 선택 슬라이더의 기본값이 낮은 쪽으로 조정
```
